# Настройка базы данных (SQLite)

## ✅ Используется SQLite (файловая база данных)

Проект использует SQLite вместо PostgreSQL. Все данные хранятся в файле `.db` в корне проекта.

## Преимущества SQLite:

- ✅ Не требует отдельного сервера БД
- ✅ Все данные в одном файле
- ✅ Легко бэкапить (просто скопировать файл)
- ✅ Работает на любом хостинге
- ✅ Идеально для продакшена без сложной инфраструктуры

## Настройка:

1. В файле `.env` укажите:
```env
DATABASE_URL="file:./dev.db"
```

2. Инициализируйте базу данных:
```bash
npm run db:generate
npm run db:push
```

3. Файл базы данных `dev.db` будет создан в корне проекта `backend-nextjs/`

## Бэкап:

Для бэкапа просто скопируйте файл `dev.db`:
```bash
cp dev.db dev.db.backup
```

## Для продакшена:

Используйте тот же SQLite файл или укажите другой путь:
```env
DATABASE_URL="file:./production.db"
```

## Важно:

- Файл `dev.db` уже добавлен в `.gitignore`
- Не коммитьте файл базы данных в git
- Для продакшена создайте отдельный файл БД
